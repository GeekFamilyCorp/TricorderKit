# MCP Policy — TricorderKit v0.7

> Core rule: **CLI = heavy work. MCP = structured access. Claude = reasoning and synthesis.**

---

## What MCP SHOULD do

- Access your back-office service (structured records, statuses, human validation)
- Read a short report generated by a CLI tool
- Update a status or trigger a whitelisted command
- Sync a validated record
- Query a short local index

## What MCP SHOULD NOT do

- Scrape websites directly
- Inject full HTML pages into context
- Replace a CLI parser
- Run arbitrary shell commands
- Access secrets without control
- Modify the vault without dry-run

---

## CLI → MCP → Claude Hierarchy

| Use case | Tool |
|---|---|
| Scrape a page / content list | CLI |
| Batch-scan sources | CLI |
| Normalize / deduplicate results | CLI |
| Read a short synthesis | Claude |
| Access your back-office | MCP |
| Export to Obsidian | CLI |
| Editorial decision | Claude |

---

## Whitelisted CLI commands (example — adapt to your domain)

```bash
python -m your_cli.cli audit sources
python -m your_cli.cli content scan-new [--source] [--type]
python -m your_cli.cli sync obsidian --dry-run
python -m your_cli.cli sync service --dry-run
python tools/your-scraper/src/your_scraper/cli.py sources audit
python tools/your-scraper/src/your_scraper/cli.py scrape source [--source]
python scripts/vault_optimizer/vault_analyzer.py --vault .
python scripts/vault_optimizer/vault_manifest.py
python scripts/validate_repo.py
python scripts/health_check.py
```

## Forbidden commands

```text
rm -rf
curl | sh / wget | sh
eval
bash -c from user input
write outside TricorderKit/
```
