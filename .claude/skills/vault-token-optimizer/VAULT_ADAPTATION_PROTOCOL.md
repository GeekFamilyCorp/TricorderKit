# VAULT_ADAPTATION_PROTOCOL.md

1. Detect vault identity: Obsidian, docs repo, source database, mixed repo.
2. Detect categories from folders, filenames, frontmatter, tags, links and templates.
3. Detect conventions: ID prefixes, dates, naming rules, index files, schemas, links.
4. Build router by category: direct read, CLI scan, MCP sync, cache, delta or exclude.
5. Generate optimization plan: commands, agents, skills, hooks, cache, dry-run, audit.
6. Apply safely only after backup/git status, dry-run report and post-change audit.
